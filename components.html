Tables

Select the appropriate table depending on the number of columns needed. Always follow these general table rules

  • Select short, precise and concise table headers
  • Text in cells should be aligned to the left
  • Numbers in cells should be aligned to the right

Information

This section deals with messages, warnings & panels for information. The overall rules for informing the users or stakeholders that uses our systems and websites:

  • If the information is narrow and regards a specific action/change, use a message/warning
  • If the information is of more general or optional nature, consider using a panel instead
  • Use clear and concise language
  • Adjust to the siutuation, e.g. IT-breach needs serious and determined language
  • Only inform and alert users when necessary

Messages

Confirmation message and neutral information are information on actions and changes with no noteworthy negative consequences. Examples are a successful download confirmation or neutral information on number of minutes left.


× Success: regards a successful positive action
× Info: regards a neutral action or change

Forms

Forms for inputs and selections should follow this section. General form rules:

  • Form labels is small & semibiold
  • Input field and areas are empty
  • Form input fields follows parent section width
  • Form selection fields follows half of the parent section width
  • Checkboxes should come before its label, so the check is evident
  • Be sure to require only necessary fields, and do not unecessary information
  • Submit buttons follows the functional button style, but takes up full form-width
